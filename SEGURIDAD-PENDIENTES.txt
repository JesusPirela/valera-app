1. CAMBIAR LAS CONTRASENAS DEBILES EXISTENTES
   La politica nueva solo aplica a contrasenas nuevas. Las cuentas viejas
   con contrasenas como "123" siguen funcionando.
   Pasos:
   a) Entrar a https://supabase.com/dashboard -> proyecto valera
   b) Authentication -> Users
   c) Identificar cuentas importantes (admins primero)
   d) Pedir a cada usuario que cambie su contrasena desde la app,
      o usar "Send password recovery" para forzar el cambio por email.

2. ROTAR LAS LLAVES DE SUPABASE
   El "service role key" y el "personal access token" han circulado por
   chats y archivos. Quien los tenga controla TODA la base de datos.
   Pasos:
   a) Dashboard -> Settings -> API -> "Reset" en service_role key
   b) https://supabase.com/dashboard/account/tokens -> borrar el token
      "sbp_..." viejo y crear uno nuevo
   c) Actualizar el secret en las Edge Functions si alguna usa la llave:
      Dashboard -> Edge Functions -> Secrets
   d) Si GitHub Actions usa estas llaves: GitHub -> repo -> Settings ->
      Secrets and variables -> Actions -> actualizar valores.
   NOTA: el "anon key" NO hay que rotarlo, es publico por diseno.

3. ACTIVAR MFA (DOBLE FACTOR) PARA LOS ADMINS
   Ya esta habilitado a nivel proyecto; falta que cada admin lo active.
   Pasos (por cada cuenta admin):
   a) Instalar Google Authenticator o similar en el celular
   b) (Requiere agregar pantalla de enrolamiento en la app, o hacerlo
      via Dashboard -> Authentication -> Users -> usuario -> MFA)
   c) Si quieren, puedo desarrollar la pantalla de MFA en la app.
